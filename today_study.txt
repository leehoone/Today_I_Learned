1. 버전관리
내가 만든 파일의 내용이 어떻게 바꼈는지 보여줌
commit을 이용해서 (누가, 언제, 무엇을) 내용을 포함하면 좀 더 버전관리가 수월함.
2. git 초기화
컴퓨터에 있는 프로젝트를 git으로 관리하겠다! 라는 선언
3. add/staging
커밋할 파일을 골라서 커밋함 
협업할 경우 겹치는 경우를 줄일수 있음
4. commit
파일의 현재 상태를 사진으로 남김 
5. commit의 역사(history)
commit한 순서대로 내역이 쌓임
6. push
로컬 repo에 있는 내용을 원격 repo에 집어넣음,원격 repo에 없는 commit을 한번에 반영함.
7. pull
원격 repo에 있는 내용을 로컬 repo로 가져옴,로컬 repo에 없는 commit을 한번에 반영함.
8. clone
초기 repo를 설정하는 과정
컴퓨터에 로컬 repo가 없을 때, 원격 repo주소를 가지고 초기 다운로드 함.
9. tracking
로컬 repo를 원격 repo에 연결
